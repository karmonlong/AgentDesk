# 🎨 多智能体工作台设计理念

> 版本：2.0  
> 设计日期：2025-11-23  
> 设计目标：打造类似 VS Code + ChatGPT 的智能体协作工作台

---

## 🎯 核心设计理念

### **整体愿景**
打造一个**专业级多智能体协作工作台**，让用户像使用 VS Code 编程一样，通过自然语言指挥多个 AI 智能体完成复杂的办公任务。

### **设计灵感**
- **首页交互**: ChatGPT / Gemini - 简洁、渐进式、引导性强
- **工作区布局**: VS Code - 侧边栏、主编辑区、面板区、多标签页
- **智能体交互**: Slack / Teams - @ 提及、频道切换、实时协作

---

## 🏠 一、首页设计（Landing Page）

### **1.1 设计目标**
- **极简主义**：空白画布，用户注意力聚焦在输入框
- **渐进式引导**：从简单到复杂，逐步展示功能
- **零学习成本**：类似 ChatGPT，一看就会用

### **1.2 视觉设计**

#### **布局结构**
```
┌─────────────────────────────────────────┐
│                                         │
│         [Logo: 🤖 Office Assistant]    │
│                                         │
│     ┌───────────────────────────────┐  │
│     │  想让我帮你做什么？           │  │
│     │  (例如：分析这份财报)    [→] │  │
│     └───────────────────────────────┘  │
│                                         │
│            [快速入口按钮区]             │
│                                         │
└─────────────────────────────────────────┘
```

#### **视觉元素**

**1. Logo & 标题**
- 位置：屏幕中央偏上
- 样式：大号字体（3rem）+ 渐变色
- 动效：淡入效果，略带呼吸感

**2. 主输入框**
- 位置：正中心
- 样式：圆角卡片（28px）+ 玻璃态效果
- 尺寸：宽度 600px（移动端自适应）
- 占位符：引导性文案，激发用户想象
- 动效：聚焦时边框发光（橙色）

**3. 快速入口**
- 位置：输入框下方
- 内容：4-6个常用场景卡片
  - 📊 分析财报
  - ✍️ 撰写报告
  - 🔍 审核文案
  - 📈 提取数据
  - 🌐 翻译文档
  - ⚖️ 合规检查
- 样式：小卡片网格布局，悬浮效果
- 交互：点击自动填入输入框

**4. 背景**
- 纯黑色（#0A0A0A）
- 径向渐变橙色光晕（中心位置，5% 透明度）
- 可选：细微的粒子动画（科技感）

### **1.3 交互流程**

#### **初始状态**
- 用户看到空白首页 + 大输入框
- 自然会想："我该输入什么？"
- 占位符和快速入口给予引导

#### **用户输入**
1. 用户输入任务描述或点击快速入口
2. 按下 Enter 或点击 → 按钮
3. **页面开始转场动画**

#### **转场动画（关键！）**
```
阶段1：收缩 (0-0.3s)
- Logo 缩小并向左上角移动
- 输入框缩小到顶部工具栏大小
- 快速入口淡出

阶段2：展开 (0.3-0.8s)
- 左侧边栏从左侧滑入
- 主工作区从中间展开
- 右侧面板从右侧滑入
- 底部状态栏从下方升起

阶段3：就绪 (0.8-1.0s)
- 智能体列表加载动画
- 对话框显示欢迎消息
- 文件资源管理器展示示例文件
```

#### **动画曲线**
- 使用 `cubic-bezier(0.16, 1, 0.3, 1)` 缓动函数
- 营造"丝滑"的感觉
- 类似 macOS / iOS 的原生动画

## 🖥️ 二、工作区设计（Workbench）

### **2.1 设计目标**
- **专业感**：像 IDE 一样强大
- **多任务并行**：支持多个工作流同时进行
- **智能体可见性**：随时知道哪些智能体在工作

### **2.2 整体布局（VS Code 风格）**

```
┌──┬───────┬──────────────────────────────┬─────────────┐
│  │       │        Header / 顶部栏       │             │
│A │   S   ├──────────────────────────────┤    Right    │
│c │   i   │                              │    Panel    │
│t │   d   │        Main Editor           │             │
│i │   e   │        (主编辑区)            │  - AI对话   │
│v │   b   │                              │  - 智能体   │
│i │   a   │    [Tab1] [Tab2] [Tab3]      │  - 工作流   │
│t │   r   │                              │             │
│y │       │                              │             │
│  │       ├──────────────────────────────┤             │
│B │       │      Panel / 面板区          │             │
│a │       │   (终端/输出/日志)           │             │
│r │       │                              │             │
└──┴───────┴──────────────────────────────┴─────────────┘
│                Status Bar / 状态栏                    │

```

## 🎭 三、交互体验设计

### **3.1 核心交互模式**

#### **模式1：快速对话**
```
用户流程：
1. 在顶部输入框输入问题
2. 系统智能路由到合适的智能体
3. 在主编辑区打开对话标签页
4. 实时显示智能体回复
5. 支持连续追问
```

#### **模式2：@ 提及指令**
```
用户流程：
1. 输入 @ 触发智能体选择器
2. 选择目标智能体（支持键盘）
3. 输入具体指令
4. 智能体开始执行
5. 右侧面板显示进度
```

#### **模式3：拖拽上传**
```
用户流程：
1. 从桌面拖拽文件到工作区
2. 文件自动上传到资源管理器
3. 弹出快速操作菜单
   - 分析文档
   - 提取数据
   - 生成摘要
   - 格式转换
4. 选择操作后自动开始处理
```

#### **模式4：多智能体协作**
```
用户流程：
1. 用户输入复杂任务："分析财报并生成报告"
2. @协调者 自动识别需要多智能体
3. 生成工作流计划：
   - 步骤1：@文档分析师 提取要点
   - 步骤2：@数据专家 分析数据
   - 步骤3：@内容创作者 撰写报告
   - 步骤4：@校对编辑 优化语言
4. 右侧面板显示工作流进度
5. 底部输出显示各智能体日志
6. 完成后在主编辑区打开结果
```

### **3.2 智能化辅助**

#### **智能建议**
- **场景建议**：根据上传的文件类型推荐场景
- **智能体推荐**：根据任务类型推荐合适的智能体
- **工作流建议**：复杂任务自动生成工作流

#### **快捷操作**
- **快捷键支持**：所有操作支持键盘快捷键
- **命令面板**：Cmd+P 打开命令面板，快速执行命令
- **最近文件**：Cmd+E 快速切换最近使用的文件

#### **智能补全**
- **@ 提及补全**：输入 @ 自动显示智能体列表
- **文件路径补全**：输入文件名自动补全
- **历史命令补全**：输入历史使用过的命令

## 🎨 四、视觉设计系统

### **4.1 色彩方案**

#### **主色调**
- **背景色系**
  - 主背景：纯黑 `#0A0A0A`
  - 次级背景：深灰 `#121212`
  - 三级背景：中灰 `#1E1E1E`
  
- **强调色系**
  - 主要强调：熔岩橙 `#FF6B00`
  - 次级强调：亮橙 `#FF8C00`
  - 渐变强调：橙→红 `linear-gradient(45deg, #FF8800, #FF6B00)`

- **文本色系**
  - 主要文本：亮灰 `#E0E0E0`
  - 次要文本：中灰 `#A0A0A0`
  - 提示文本：暗灰 `#666666`

- **状态色系**
  - 成功：翠绿 `#10B981`
  - 警告：金黄 `#F59E0B`
  - 错误：红色 `#EF4444`
  - 信息：蓝色 `#3B82F6`

#### **智能体专属色**
```
📊 文档分析师 → 蓝色系 #3B82F6
✍️ 内容创作者 → 紫色系 #8B5CF6
📈 数据专家 → 绿色系 #10B981
✅ 校对编辑 → 青色系 #06B6D4
🌐 翻译专家 → 粉色系 #EC4899
⚖️ 合规官 → 黄色系 #F59E0B
🎯 协调者 → 橙色系 #FF6B00
```

## 🎮 五、场景化工作流

### **5.1 投研分析场景**

#### **用户故事**
> 作为一名投研分析师，我需要快速分析一份季度财报，提取关键数据并生成投研报告。

#### **工作流程**
```
1. 用户进入工作台，选择"投研分析"场景
   
2. 拖拽财报 PDF 到资源管理器
   → 系统识别文件类型
   → 弹出快速菜单：
      - 提取关键数据
      - 生成数据图表
      - 撰写分析报告
   
3. 用户选择"提取关键数据"
   → @文档分析师 自动开始工作
   → 右侧面板显示进度
   → 底部输出显示日志
   
4. 数据提取完成，自动打开数据标签页
   → 用户查看提取的数据表格
   → 点击"生成图表"按钮
   
5. @数据专家 生成趋势图
   → 在主编辑区显示图表
   → 支持交互式调整
   
6. 用户输入："基于这些数据撰写分析报告"
   → @协调者 识别为多步骤任务
   → 自动编排工作流：
      步骤1：@内容创作者 撰写报告初稿
      步骤2：@校对编辑 优化语言
      步骤3：@合规官 审核合规性
   
7. 工作流自动执行
   → 右侧显示实时进度
   → 各智能体依次完成任务
   
8. 最终报告生成
   → 在新标签页打开
   → 用户可继续编辑
   → 支持导出多种格式
```

### **5.2 合规审核场景**

#### **用户故事**
> 作为一名合规专员，我需要审核多份营销文案，确保符合监管要求。

#### **工作流程**
```
1. 用户切换到"合规审核"场景
   → 智能体优先级自动调整
   → @合规官 置顶
   
2. 批量上传营销文案文件
   → 文件列表显示在资源管理器
   → 每个文件显示状态标签：待审核
   
3. 用户选择第一份文案
   → 自动在主编辑区打开
   → 右侧显示合规检查清单
   
4. 用户点击"快速审核"
   → @合规官 开始自动审核
   → 实时高亮标注问题区域：
      - ❌ 违规词汇（红色）
      - ⚠️ 风险提示不足（黄色）
      - 💡 优化建议（蓝色）
   
5. 审核完成，生成审核报告
   → 问题列表显示在右侧
   → 每个问题可展开查看详情
   → 提供修改建议
   
6. 用户点击"自动修复"
   → @内容创作者 根据合规建议修改文案
   → 实时显示修改对比
   
7. 修改完成，再次审核
   → @合规官 复审
   → 通过后标记为"✅ 合规"
   
8. 批量处理剩余文件
   → 点击"批量审核"
   → 所有文件依次自动审核
   → 生成汇总报告
```

### **5.3 翻译本地化场景**

#### **用户故事**

```
1. 用户切换到"翻译本地化"场景

   → 系统自动识别语言：简体中文
   
3. 用户输入："翻译成英文，保持专业术语准确"

   → 右侧显示翻译进度
   
4. 翻译完成，显示双语对照视图
   ┌──────────────┬──────────────┐
   │   中文原文   │   英文译文   │
   ├──────────────┼──────────────┤
   │ 产品特性     │ Features     │
   │ ...          │ ...          │
   └──────────────┴──────────────┘
   
5. 用户发现术语翻译不准确
   → 点击术语，选择"修正翻译"
   → 输入正确的专业术语
   → 系统自动更新所有相关位置
   
6. 用户输入："检查翻译质量并优化"
   → @校对编辑 审核译文
   → 提出优化建议
   → 逐句显示改进方案
   
7. 用户确认修改
   → 译文实时更新
   → 生成最终翻译版本
   
8. 导出双语对照表
   → 支持多种格式：Word, PDF, Excel
```
