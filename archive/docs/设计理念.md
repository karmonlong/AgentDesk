# 📋 首页与工作台功能定位分析

> 版本：1.0  
> 分析日期：2025-11-23  
> 目的：明确首页（Landing Page）与工作台（Workbench）的功能边界

---

## 🎯 一、首页（Landing Page）的核心定位

### **1.1 用户最希望在首页实现什么？**

#### **用户心理状态模型**

```
用户打开应用的第一瞬间：
"我想快速完成一个任务"
         ↓
"我不想学习复杂的操作"
         ↓
"我希望直接说出我的需求"
         ↓
"系统能自动帮我处理"
```

---

### **1.2 首页应该提供的核心功能**

#### **🎯 功能1：零障碍的任务发起**

**场景描述**：
- 用户脑子里有个明确的任务
- 需求："我想让系统帮我做XXX"

**用户期望**：
- 看到大输入框，直接输入自然语言
- 系统自动识别任务类型
- 自动调用合适的智能体
- 自动展开工作台开始执行

**示例交互**：
```
用户输入：
"分析这份财报的核心数据"

系统响应：
  ✅ 识别任务类型：数据分析
  ✅ 自动启动工作台
  ✅ 弹出文件上传提示
  ✅ 调用 @文档分析师 + @数据专家
  ✅ 开始执行
```

---

#### **🎯 功能2：快速场景切换**

**场景描述**：
- 用户有固定的工作场景
- 需求："我经常做XXX类型的工作"

**实现方式**：
- 首页显示常用场景卡片
- 点击自动进入预配置的工作台
- 智能体优先级已调整好
- 快速入口已准备就绪

**示例场景卡片**：

└─────────────────────┘

┌─────────────────────┐
│ ✍️ 内容创作         │
│ 撰写报告和邮件      │
│ [进入工作台]        │
└─────────────────────┘

┌─────────────────────┐
│ 📈 数据处理         │
│ 提取和分析数据      │

└─────────────────────┘
```

#### **🎯 功能3：快速历史恢复**

**场景描述**：
- 用户继续上次未完成的工作

#### **🎯 功能4：探索式引导**

**场景描述**：
- 新用户不知道能做什么
- 需求："这个系统能帮我做什么？"

**实现方式**：
- **占位符文案引导**
  ```
  "想让我帮你做什么？"
  "例如：分析这份财报"
  "例如：撰写一封感谢邮件"
  "例如：检查这段文案的合规性"
  ```

- **快速入口展示典型任务**
  - 每个场景卡片有清晰的说明
  - 悬浮显示更多信息

- **智能建议**
  ```
  💡 基于你的角色，我们推荐：
  - 投研分析师 → 投研分析场景
  - 合规专员 → 合规审核场景
  - 内容运营 → 内容创作场景
  ```

## 🔄 二、首页与工作台的分工

### **2.1 首页的职责（Landing Page）**

#### **核心定位**
> **任务发起器 + 场景选择器 + 历史管理器**

```
┌────────────────────────────────────┐
│         首页（Landing Page）       │
│                                    │
│  ✅ 职责：                         │
│  • 快速输入任务意图                │
│  • 选择预设工作场景                │
│  • 恢复历史工作会话                │
│  • 零学习成本的入口                │
│  • 快速示例和引导                  │
│  • 智能场景推荐                    │
│                                    │
│  ❌ 不做：                         │
│  • 复杂的文件管理                  │
│  • 详细的任务执行                  │
│  • 多轮复杂对话                    │
│  • 智能体状态监控                  │
│  • 工作流程可视化                  │
│  • 结果编辑和导出                  │
└────────────────────────────────────┘
```

#### **设计哲学**
> "让用户像和朋友聊天一样，轻松说出自己想做什么，然后系统自动准备好专业工具"

#### **交互流程**
```
第1步：首页初始状态
┌─────────────────────────────────┐
│  空白画布 + 大输入框             │
│  Logo 居中显示                   │
│  场景卡片网格布局                │
│  历史会话列表（如有）            │
└─────────────────────────────────┘
         ↓
第2步：用户输入任务
┌─────────────────────────────────┐
│  用户输入："帮我分析这份财报"   │
│  或点击场景卡片："投研分析"     │
│  或点击历史："继续昨天的工作"   │
└─────────────────────────────────┘
         ↓
第3步：系统智能判断
┌─────────────────────────────────┐
│  解析任务类型：数据分析          │
│  需要智能体：                    │
│    - @文档分析师                 │
│    - @数据专家                   │
│  需要面板：                      │
│    - 文件上传区                  │
│    - 对话窗口                    │
│    - 数据可视化                  │
└─────────────────────────────────┘
         ↓
第4步：转场动画（0.8秒）
┌─────────────────────────────────┐
│  阶段1（0-0.3s）：收缩           │
│    - Logo 缩小移至左上角         │
│    - 输入框缩小至顶部            │
│    - 场景卡片淡出                │
│                                  │
│  阶段2（0.3-0.8s）：展开         │
│    - 左侧边栏滑入                │
│    - 主工作区展开                │
│    - 右侧面板滑入                │
│    - 底部状态栏升起              │
│                                  │
│  阶段3（0.8-1.0s）：就绪         │
│    - 智能体列表加载              │
│    - 文件区域准备就绪            │
│    - 对话框显示欢迎              │
└─────────────────────────────────┘
         ↓
第5步：工作台就绪
┌─────────────────────────────────┐
│  用户可以立即开始工作：          │
│  - 上传文件                      │
│  - 与智能体对话                  │
│  - 查看实时进度                  │
└─────────────────────────────────┘
```

### **2.2 工作台的职责（Workbench）**

#### **核心定位**
> **专业协作空间 + 任务执行中心 + 多智能体指挥台**

```
┌────────────────────────────────────┐
│         工作台（Workbench）        │
│                                    │
│  ✅ 职责：                         │
│  • 多文件管理和预览                │
│  • 多智能体并行协作                │
│  • 复杂多轮对话                    │
│  • 工作流可视化                    │
│  • 实时状态监控                    │
│  • 结果编辑和输出                  │
│  • 历史记录查看                    │
│  • 任务暂停和恢复                  │
│  • 多标签页管理                    │
│                                    │
│  ❌ 不做：                         │
│  • 新手引导（已在首页完成）        │
│  • 场景选择（已在首页完成）        │
│  • 初始任务输入（已在首页完成）    │
└────────────────────────────────────┘

#### **设计哲学**
> "像 Cursor/VS Code 一样专业，像 Slack 一样协作，像 ChatGPT 一样自然"

### **3.2 状态转换逻辑**

#### **首页 → 工作台的触发条件**

**触发1：用户输入任务描述并提交**
```
条件：
- 用户在输入框输入文本
- 按 Enter 或点击提交按钮

动作：
1. 系统解析任务意图
2. 识别需要的智能体
3. 配置工作台布局
4. 执行转场动画
5. 工作台就绪
```

**触发2：用户点击场景卡片**
```
条件：
- 用户点击"投研分析"等场景卡片

动作：
1. 加载场景预设配置
2. 激活相关智能体
3. 设置智能体优先级
4. 执行转场动画
5. 工作台就绪（场景模式）
```

**触发3：用户点击历史会话**
```
条件：
- 用户点击历史会话的"继续工作"

动作：
1. 加载历史会话数据
2. 恢复文件列表
3. 恢复对话历史
4. 恢复智能体状态
5. 执行转场动画
6. 工作台就绪（恢复模式）
```

---

#### **工作台 → 首页的触发条件**

**触发1：用户点击顶部 Logo**
```
条件：
- 用户点击左上角 Logo / "返回首页"

动作：
1. 弹出确认对话框：
   "是否保存当前工作？"
   [保存并返回] [直接返回] [取消]
   
2. 如果选择保存：
   - 保存当前状态为历史会话
   - 执行反向转场动画
   - 返回首页
   
3. 如果选择直接返回：
   - 执行反向转场动画
   - 返回首页
```

**触发2：用户关闭所有标签页**
```
条件：
- 用户关闭最后一个工作标签页

动作：
1. 弹出提示：
   "所有任务已完成，是否返回首页？"
   [返回首页] [继续工作]
   
2. 如果选择返回：
   - 保存会话记录
   - 执行反向转场动画
   - 返回首页
```

**触发3：用户点击"新任务"**
```
条件：
- 用户点击顶部的"新任务"按钮

动作：
1. 如果当前有未完成任务：
   - 提示是否保存
   
2. 执行反向转场动画
3. 返回首页（准备新任务）
```

### **6.2 核心设计原则**

**原则1：职责分离**
> 首页专注于"想做什么"，工作台专注于"怎么做"

**原则2：渐进式体验**
> 从简单到复杂，从引导到专业，逐步展开功能

**原则3：零学习成本**
> 新用户看到首页就知道怎么用，老用户可以快速恢复工作

**原则4：无缝衔接**
> 转场动画让两个页面自然过渡，体验连贯流畅

**原则5：状态保持**
> 任务状态在首页和工作台之间无缝传递和保存
