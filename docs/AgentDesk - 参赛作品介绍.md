# AgentDesk - AI 竞赛路演逐字稿提纲

## 1. 开场：痛点与愿景 (The Hook)
- **背景**：资管行业面临“信息过载”与“容错率低”的双重挑战。
- **痛点**：传统的 AI 工具（如 ChatPDF）功能单一，缺乏专业深度，且无法处理复杂的长链条任务。
- **愿景**：我们需要的不只是一个聊天机器人，而是一个**“懂业务的智能体团队”**。
- **引出产品**：AgentDesk —— 基于 LangGraph 构建的资管智能体工作台。

## 2. 核心理念：从“工具”到“团队” (The Concept)
- **核心差异化**：展示 AgentDesk 与传统 AI 的区别。
- **拟人化分工**：介绍 8 个专业智能体（文档分析师、数据专家、合规官等）。
- **交互创新**：演示类似 IDE 的 `@提及` 交互模式，用户是在“指挥团队”而非“操作软件”。

## 3. 关键功能演示 (Feature Walkthrough)
- **场景一：深度文档作业（Deep Work）**
    - 展示从“上传财报”到“自动提取数据”再到“生成研报”的全流程。
    - 强调 **LangGraph** 带来的状态管理能力（断点续传、人工审核）。
- **场景二：多智能体协作（Collaboration）**
    - 演示 `@文档分析师` 提取信息后，无缝流转给 `@内容创作者` 写邮件，再由 `@翻译专家` 翻译。
- **场景三：可信知识管理（Trustworthy RAG）**
    - 展示基于 ChromaDB 的语义检索。
    - 强调“精准引用”功能，解决 AI 幻觉问题，满足金融合规需求。
- **场景四：多模态创新（The "Wow" Factor）**
    - 展示 **PDF 转播客** 功能（Open NotebookLM），将枯燥研报转化为双人对话音频，利用碎片时间学习。

## 4. 技术架构与优势 (Technical Architecture)
- **技术栈**：LangGraph (编排) + Kimi LLM (长文本/推理) + FastAPI + ChromaDB。
- **稳定性**：强调状态机（State Machine）带来的确定性和可控性。
- **体验**：介绍“玻璃拟态”UI 设计与极速响应，打造 Premium 级工作台体验。

## 5. 结语：价值与未来 (Closing)
- **总结价值**：提效（Efficiency）、准确（Accuracy）、专业（Professionalism）。
- **升华**：AgentDesk 重新定义了人机协作模式——人类是“指挥官”，AI 是“执行官”。
